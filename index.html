Reflectie

AIM-reflectie

Tijdens de afgelopen periode van WPL1 heb ik de kans gekregen om de fundamenten van software development te verkennen. De focus lag op het vertalen van theoretische kennis naar praktische vaardigheden, waarbij we werkten met tools zoals C#, HTML/CSS en versiebeheer via GitHub.

Een bijzonder positief element van deze periode was het werken met HTML/CSS en de gastspreken-sessies. Het was itressant om te zien hoe de code die ik schrijf impact heeft op een UI. Vooral de sessies met mensen uit het werkveld gaven me een positief inzicht.

Minder leuk vond ik de momenten waarop ik vastliep tijdens het debuggen van complexe lussen of de styling in CSS. Het was soms frustrerend wanneer de lay-out niet meewerkte of wanneer een bug na lang zoeken een klein typefoutje bleek te zijn. Deze momenten van "vastzitten" waren leerrijk, maar vergden op dat moment veel geduld.

Ik neem waardevolle nieuwe inzichten mee voor het vervolg van mijn studies. Het belangrijkste inzicht is dat code eigenlijk nooit "af" is, feedback en kritisch kijken naar je eigen werk zijn essentieel om een betere programmeur te worden.

Mijn grootste persoonlijke succes was het succesvol bouwen van mijn eigen portfolio en CV in HTML/CSS. Ik ben er trots op dat ik technische vaardigheden heb kunnen combineren met een eigen ontwerp. Het geeft veel voldoening om een functioneel eindproduct te zien dat mijn eigen groei van de afgelopen weken weerspiegelt.
